## Phase 0 — Model-lifecycle safety net (v2.4.0)

### Motivation

Every LLM call in the Curator flows through `generateText()` in [`src/brain/llm.js`](../src/brain/llm.js). The pinned default model IDs (`gemini-2.5-flash-lite`, `claude-sonnet-4-6`) are provider-controlled strings: when a provider retires or renames one of them, every Curator installation pinned to that ID starts returning 404 and the core features (ingest, chat, upcoming AI-Health) fail silently for non-technical users.

This audit phase builds a safety net so existing installations survive provider deprecations until the next Check-for-Updates cycle.

### Scope audit (files read, no changes)

Files touched by Phase 0:
- `src/brain/llm.js` — adds fallback chain logic
- `src/routes/config.js` — one new field in `/api/config/api-keys` response
- `src/public/app.js` — one new function `renderFallbackBanner()`
- `src/public/styles.css` — one new class `.settings-fallback-banner`

Files audited but NOT modified:
- `src/brain/ingest.js` — calls `generateText()` with `'json'` response format. Confirmed that JSON prompting works on both Gemini (native `responseMimeType`) and Anthropic (prompt instruction + `parseJSON` with `jsonrepair`). Fallback chain is transparent to callers.
- `src/brain/chat.js`, `src/brain/query.js` — text-mode `generateText()` callers. No changes needed.
- `src/brain/files.js`, `src/brain/sync.js`, `src/brain/health.js`, `src/brain/config.js` — no LLM calls; untouched.
- `mcp/` — read-only; doesn't use `llm.js`.
- `src/routes/*.js` (excluding config.js) — unchanged.

This means: ingest, chat, health (algorithmic fixes), sync, MCP, updater, installer — none of their code paths are modified. Only the common LLM dispatch layer gains a fallback wrapper.

### Design decisions

**1. Fallback only on `model-not-found`.** Rate limits (429) and service-unavailable (503) are NOT retried via the fallback chain — those already have dedicated exponential-backoff handling in the existing `generateText()` retry loop. Auth errors (401) and permission errors (403) propagate immediately as before. This keeps the chain's semantics crisp: "the pinned model no longer exists."

**2. Error detection via message + status matching.** Anthropic throws a typed `NotFoundError` with `status: 404`; Gemini returns generic errors with messages like `"[404 Not Found] models/X is not found for API version Y"`. The `isModelNotFound()` helper matches both via a combination of status code, the literal `"404"` substring + `"not found"`/`"is not supported"` keywords, and the Anthropic-specific `"not_found_error"` marker. Tested successfully against the Gemini case; Anthropic case covered by the typed `status === 404` short-circuit.

**3. Module-level fallback state, not per-request.** A single `_activeFallback` variable tracks the most recent fallback event. It's set when a fallback succeeds, cleared when the primary succeeds again. This is per-process (lost on restart) which is correct — on restart, the server retries the primary first and the state re-establishes itself from reality.

**4. Gemini and Anthropic chains are separate.** Different generations of models with different pricing/capability profiles; mixing them would make "Active model" unreadable.

**5. Settings UI surfaces fallback prominently but unobtrusively.** An amber banner below the provider badge, text tells the user what's happening and where to click to update. No modal, no blocking — the app still works, the banner is information + call-to-action.

### Implementation summary

```js
// llm.js additions (strictly additive — existing callLLM refactored to callProvider + fallback loop)
const FALLBACK_CHAINS = {
  gemini: ['gemini-2.5-flash', 'gemini-1.5-flash', 'gemini-1.5-flash-latest'],
  anthropic: ['claude-sonnet-4-5', 'claude-3-7-sonnet-latest', 'claude-3-5-sonnet-latest'],
};
let _activeFallback = null;
export function getFallbackStatus() { return _activeFallback; }
function isModelNotFound(err) { /* status 404 + message matching */ }
async function callProvider(provider, model, ...) { /* was the body of callLLM */ }
async function callLLM(...) {
  const chain = [primary, ...FALLBACK_CHAINS[provider]];
  for (let i = 0; i < chain.length; i++) {
    try {
      const result = await callProvider(provider, chain[i], ...);
      if (i === 0) clearFallbackForProvider(provider);
      else setFallback(chain[0], chain[i]);
      return result;
    } catch (err) {
      if (isModelNotFound(err) && i < chain.length - 1) continue;
      throw err;
    }
  }
}
```

Config endpoint: `/api/config/api-keys` now includes `fallback: getFallbackStatus()`. Frontend reads it and renders an amber banner if populated.

### Anthropic pipeline verification

The user reports never having tested Anthropic in production. The code path was statically audited:

1. **Model ID in `DEFAULTS`:** `claude-sonnet-4-6`. The bundled SDK v0.39.0 `Model` type lists up to `claude-3-7-sonnet-latest` explicitly, but accepts arbitrary strings via `(string & {})` — so the newer ID will pass build and runtime *if Anthropic's API accepts it*. If it 404s, the fallback chain catches it with `claude-sonnet-4-5` → `claude-3-7-sonnet-latest` → `claude-3-5-sonnet-latest`.
2. **JSON response handling:** Anthropic has no `responseMimeType` equivalent. The `ingest.js` prompts instruct *"Return ONLY valid JSON"* explicitly; downstream, `parseJSON()` with `jsonrepair` absorbs minor deviations. Functionally equivalent but less deterministic than Gemini's native JSON mode.
3. **Error detection:** Anthropic SDK throws typed `NotFoundError` with `status: 404`. The `isModelNotFound()` helper catches this via the explicit `err.status === 404` branch.

Result: Anthropic pipeline is structurally sound; the one caveat is that JSON-producing calls (ingest) may occasionally require the `jsonrepair` safety net more often than with Gemini. This is noted in `docs/model-lifecycle.md`.

## Phase 0 Addendum — Anthropic low-cost default (v2.4.1)

### Motivation

v2.4.0 shipped with `DEFAULTS.anthropic = 'claude-sonnet-4-6'` — Sonnet being Anthropic's *mid-tier* model. For wiki ingestion of large libraries, this mis-matched the cost profile of the Gemini default (`gemini-2.5-flash-lite`, which is explicitly the low-cost tier). The user clarified that the Anthropic default should be the low-cost tier too.

### Change

- `DEFAULTS.anthropic`: `claude-sonnet-4-6` → **`claude-haiku-4-5`**
- `FALLBACK_CHAINS.anthropic` reordered to prefer Haiku variants first (same cost tier), then escalate to Sonnet family only if the entire Haiku line is retired:
  - `claude-3-5-haiku-latest` — previous Haiku gen, SDK-typed, definitely works
  - `claude-3-5-haiku-20241022` — explicit stable version as last-resort Haiku
  - `claude-sonnet-4-5` — quality upgrade tier
  - `claude-3-7-sonnet-latest` — rolling alias
  - `claude-3-5-sonnet-latest` — deep fallback

### Opt-out for users who want Sonnet

Documented in `docs/model-lifecycle.md`:

```bash
# in .env — if you want higher-quality Anthropic for dense content
LLM_MODEL=claude-sonnet-4-5
```

The fallback chain still applies on top of the override.
